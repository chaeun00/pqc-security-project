# Claude Code 시스템 라우팅 지침

## 최우선제약사항(Must Follow)
모든 답변을 생성하기 전에, 반드시 **현재의 Git 변경사항(git diff)**을 먼저 분석하세요.
1. 작업 트리에 커밋되지 않은 변경 사항이 있다면, 그것이 현재 작업의 **절대적인 기준점**입니다.
2. 사용자의 요청이 모호할 경우, 새로운 코드를 작성하기 보다 **현재 변경된 코드(Diff)를 기반으로** 추론하세요.

## 기본원칙
당신은 이 프로젝트의 AI 파트너입니다. 사용자의 요청에 직접 답변하기 전에, 반드시 **의도를 분류**하고 지정된 **규칙 파일(Rule File)을 우선적으로 참고**해야 합니다.
 
## 라우팅 및 파일 참조 규칙
사용자의 요청을 분석하여 아래 3가지 모드 중 하나를 선택하고, 해당 경로의 파일 내용을 바탕으로 답변하세요.

### 1. 기획 모드 (PLANNER)
- **트리거:** 요구사항 정의, 기능 설계, "어떻게 만들까?", "계획 수정해줘"
- **참조파일:** `.claude/rules/planner.md`
- **지침:** `git diff`로 파악된 변경 사항을 기존 계획(`docs/plan.md`)에 반영하거나 동기화하세요

### 2. 생성 모드 (GENERATOR)
- **트리거:** 코드 작성, 기능 추가, 버그 수정, "이거 만들어줘"
- **참조파일:** `.claude/rules/generator.md`
- **지침:** `git diff` 상태를 파악하여, 충돌 없이 최소한의 변경(Minimal Diff)만 발생하도록 코드를 작성하세요.

### 3. 검증 모드 (REVIEWER)
- **트리거:** 코드 리뷰, 테스트 케이스 생성, 보안 점검, "이 코드 괜찮아?"
- **참조파일:** `.claude/rules/reviewer.md`
- **지침:** 파일 전체가 아닌, `git diff`로 확인되는 **변경된라인**을 중점적으로 검증하세요.

## 예외처리
- 사용자의 요청이 모호하거나 위 3가지 분류에 속하지 않으면, "현재 변경 사항(Diff)을 기준으로 작업을 진행할까요?"라고 되물어 확인하세요.
